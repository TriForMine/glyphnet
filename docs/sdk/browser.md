# Browser SDK

The browser SDK is expected to wrap `glyphnet-wasm`.

Planned capabilities:

- SVG and PNG generation from JavaScript or TypeScript.
- Camera scanning through `MediaDevices.getUserMedia`.
- Burst-mode display and receive loops synchronized with `requestAnimationFrame`.
- Web Worker offload for decode and CV work.
- Optional WebGPU acceleration for thresholding and rectification.

Current scaffold:

- `sdk/browser/package.json`
- `sdk/browser/src/index.ts`
- `demos/browser/index.html`
- `demos/debug/index.html`

Current WASM bridge:

- `encodeSvg(input)` returns an SVG document.
- `encodeSvgAuthenticated(input, authKeyHex, keyId)` returns an SVG document
  with an embedded authenticity envelope.
- `encodeSvgWithGeometry(input, modulePx, quietZoneModules)` returns SVG with
  explicit render geometry.
- `encodePngWithGeometry(input, modulePx, quietZoneModules)` returns PNG bytes
  generated by the Rust raster renderer.
- `encodePngWithLayoutGeometryAuthenticated(input, authKeyHex, keyId, layout, modulePx, quietZoneModules)`
  returns PNG bytes with an embedded authenticity envelope.
- `scanRgbaJson(rgba, width, height, mode)` runs the Rust still-image scanner
  and returns JSON diagnostics.
- `scanRgbaJsonWithVerification(rgba, width, height, mode, verifyKeyHex, verifyKeyId)`
  scans and verifies authenticity if the payload is enveloped.
- `BurstScanSession.withConfigAndVerification(mode, maxFrames, verifyKeyHex, verifyKeyId)`
  enables authenticity verification on completed burst payloads.
- `signDetachedAuth(input, authKeyHex, keyId)` returns detached signature JSON:
  `{"key_id": <u32>, "payload_len": <u32>, "tag_hex": "<32 hex chars>"}`.
- `verifyDetachedAuth(input, signatureJson, keyringJson)` verifies detached
  signature JSON against keyring JSON and returns:
  `{"verified": <bool>, "key_id": <u32>, "error": <string|null>}`.

Keyring JSON format for detached verification:

```json
[
  { "key_id": 1, "key_hex": "001122...ff" },
  { "key_id": 42, "key_hex": "aabbcc...dd" }
]
```

Debug workbench:

Build `glyphnet-wasm`, serve the repository root, then open
`/demos/debug/index.html` to import PNG/JPEG images and inspect the real Rust
scanner output, crop attempts, estimated quads, and JSON diagnostics. The
debugger intentionally does not duplicate scanner heuristics in JavaScript.
Image import, sample generation, manual crop selection, and overlay rendering
are browser-side; detection and decoding are Rust/WASM.

```powershell
wasm-pack build crates/glyphnet-wasm --target web --out-dir ../../sdk/browser/pkg
python -m http.server 8765 --bind 127.0.0.1
```
